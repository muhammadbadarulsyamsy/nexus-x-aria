# Dev.to Risk Notes

## Purpose

These notes explain what must be avoided when publishing the Dev.to article.

## Main Risk

The article must not make ARIA look like a guaranteed security product.

ARIA is:

```text
a transparent heuristic educational MVP
```

ARIA is not:

```text
a certified scam detector
a financial advisor
an investment tool
an enterprise security product
a guarantee of wallet safety
```

## Claim Control

Use:

```text
helps identify common risk signals
can make red flags easier to see
educational heuristic scoring
transparent rules
```

Avoid:

```text
detects all scams
prevents phishing
protects your wallet
guarantees safety
proves an airdrop is legit
```

## Crypto Safety

Do not encourage:

- trading
- speculation
- airdrop farming for profit
- connecting real wallets
- approving unknown contracts
- signing unknown messages

Encourage:

- burner wallet
- link verification
- reading before signing
- not sharing secrets
- independent research

## Platform Safety

Do not publish if the platform asks for:

- billing
- credit card
- paid upgrade
- token
- API key
- private key
- seed phrase
- password
- OTP
- cookie

## Tone

The tone should be:

- calm
- technical but readable
- educational
- transparent
- modest

Not:

- hype
- salesy
- fearmongering
- financial
- overconfident
