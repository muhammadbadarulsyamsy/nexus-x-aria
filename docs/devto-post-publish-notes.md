# Dev.to Post-Publish Notes v0.1

Use this after publishing the article manually.

## Immediate Actions

- [ ] Save the article URL.
- [ ] Confirm article displays correctly.
- [ ] Check links again.
- [ ] Confirm disclaimer is visible.
- [ ] Confirm limitations are visible.

## What Not To Do

Do not:

- spam the article link
- DM people aggressively
- post repeatedly in unrelated communities
- claim guaranteed scam protection
- claim ARIA is a financial tool
- promise that ARIA prevents wallet theft

## Acceptable Sharing

Acceptable low-risk sharing:

- add the link to a local project note
- optionally add it to GitHub README later
- share only in relevant places where self-posting is allowed
- share as education, not sales

## Feedback Handling

If someone criticizes ARIA:

- do not argue aggressively
- note the criticism
- check if it reveals a real limitation
- consider adding future docs or test cases
- remember that ARIA is an MVP

## If a Serious Issue Is Found

If someone reports a serious bug or unsafe claim:

1. Do not deny immediately.
2. Reproduce the issue if possible.
3. Add it to notes.
4. Fix docs or scoring if needed.
5. Do not make a new release until verified.
