# v0.2.0 Response Schema Draft

## Purpose

This document defines the planned response shape for `POST /score-airdrop` in NEXUS-X ARIA v0.2.0.

This is a design draft only.

## Required Top-Level Fields

Existing fields:

```text
service
version
risk_score
risk_level
verdict
red_flags
safe_actions
summary
```

New v0.2.0 fields:

```text
rule_hits
score_breakdown
input_quality
confidence_note
false_positive_notes
```

## Field Definitions

### service

Type:

```text
string
```

Example:

```json
"service": "nexus-x-aria"
```

### version

Type:

```text
string
```

Example:

```json
"version": "0.2.0"
```

### risk_score

Type:

```text
number
```

Range:

```text
0 - 100
```

### risk_level

Type:

```text
string
```

Allowed values:

```text
low
medium
high
critical
```

### verdict

Type:

```text
string
```

Suggested values:

```text
likely_safe
caution
avoid
```

Existing runtime wording may be preserved if different, but should remain documented.

### red_flags

Type:

```text
array<string>
```

Purpose:

```text
Human-readable risk findings.
```

### safe_actions

Type:

```text
array<string>
```

Purpose:

```text
Human-readable safe next steps.
```

### summary

Type:

```text
string
```

Purpose:

```text
Short natural-language summary of the analysis.
```

### rule_hits

Type:

```text
array<string>
```

Purpose:

```text
Machine-friendly rule IDs triggered by the analysis.
```

Example:

```json
[
  "WALLET_CONNECT_REQUEST",
  "SIGN_MESSAGE_REQUEST"
]
```

### score_breakdown

Type:

```text
array<object>
```

Each item:

```json
{
  "rule_id": "SIGN_MESSAGE_REQUEST",
  "category": "signing_request",
  "points": 30,
  "severity": "high",
  "reason": "Required tasks mention signing a message."
}
```

Required item fields:

```text
rule_id
category
points
severity
reason
```

### input_quality

Type:

```text
object
```

Shape:

```json
{
  "level": "partial",
  "missing_fields": ["token_contract"],
  "notes": [
    "The score may be less reliable because token_contract is missing."
  ]
}
```

Required fields:

```text
level
missing_fields
notes
```

Level values:

```text
complete
partial
weak
```

### confidence_note

Type:

```text
string
```

Recommended exact value:

```text
This result is based on heuristic text and metadata analysis only. It may contain false positives or false negatives.
```

### false_positive_notes

Type:

```text
array<string>
```

Purpose:

```text
Human-readable notes when ARIA intentionally avoids a false positive.
```

Example:

```json
[
  "Educational approval warning detected; approval language was not penalized as risky instruction."
]
```

## Example Full Response

```json
{
  "service": "nexus-x-aria",
  "version": "0.2.0",
  "risk_score": 85,
  "risk_level": "critical",
  "verdict": "avoid",
  "red_flags": [
    "Requires wallet connection",
    "Requires signing a message or transaction",
    "No token contract provided"
  ],
  "safe_actions": [
    "Use a burner wallet for airdrops",
    "Verify official links through multiple credible sources",
    "Do not share private keys or seed phrases"
  ],
  "summary": "The project contains multiple high-risk airdrop signals.",
  "rule_hits": [
    "WALLET_CONNECT_REQUEST",
    "SIGN_MESSAGE_REQUEST",
    "MISSING_TOKEN_CONTRACT"
  ],
  "score_breakdown": [
    {
      "rule_id": "WALLET_CONNECT_REQUEST",
      "category": "wallet_request",
      "points": 20,
      "severity": "medium",
      "reason": "Required tasks mention wallet connection."
    },
    {
      "rule_id": "SIGN_MESSAGE_REQUEST",
      "category": "signing_request",
      "points": 30,
      "severity": "high",
      "reason": "Required tasks mention signing a message."
    }
  ],
  "input_quality": {
    "level": "partial",
    "missing_fields": [
      "token_contract"
    ],
    "notes": [

    ]
  },
  "confidence_note": "This result is based on heuristic text and metadata analysis only. It may contain false positives or false negatives.",
  "false_positive_notes": []
}
```
