# Manual Publication Test Plan v0.1

## Purpose

This document defines a safe, free, manual publication test for NEXUS-X ARIA.

The goal is to look for interest signals, not to monetize immediately.

This is a docs-only planning milestone.

## Current Product Position

```text
NEXUS-X ARIA is a lightweight serverless API that helps developers inspect common Web3 airdrop risk signals with explainable heuristic scoring.
```

## Why This Exists

ARIA is technically ready as an MVP, but it is not yet market-validated.

The next useful step is to see whether external developers, educators, or Web3 builders understand and care about the project.

## What This Test Should Prove

```text
Can people understand ARIA quickly?
Does anyone click, star, comment, or ask about it?
Does the public value message work?
Is the README/article/listing copy clear enough?
```

## What This Test Does Not Prove

```text
revenue
product-market fit
security guarantee
consumer demand
paid API demand
```

## Test Style

```text
manual
free
low-risk
no billing
no card
no API key
no paid platform
no automation spam
no aggressive outbound messages
```

## Candidate Free Channels

Use only channels that can be accessed safely and manually.

```text
GitHub repository visibility and README
Dev.to article draft/manual post
personal/social short post if user already has account
developer/community share only where self-promotion is allowed
local Postman collection reference from repo
```

Do not publish on a platform if it asks for billing, card, upgrade, payout setup, wallet connection, API token, secret, password, OTP, or cookie sharing.

## Suggested Test Window

```text
7 to 14 days
```

## Suggested Test Order

```text
1. Confirm GitHub README/public message is clear.
2. Publish or prepare one Dev.to/manual article.
3. Share one short, non-spam post if there is a safe existing channel.
4. Wait and record interest signals.
5. Review whether to continue, pause, or pivot.
```
