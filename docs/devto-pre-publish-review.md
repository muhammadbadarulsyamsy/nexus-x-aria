# Dev.to Pre-Publish Review v0.1

Use this checklist before publishing the NEXUS-X ARIA article.

## Article Identity

- [ ] Article file is `docs/devto-article-polished.md`.
- [ ] Title is educational, not hype-based.
- [ ] Article describes ARIA as a heuristic educational MVP.
- [ ] Article does not describe ARIA as a complete security product.

## Safety Claims

Allowed:

- [ ] “helps identify common risk signals”
- [ ] “transparent heuristic scoring”
- [ ] “educational and informational”
- [ ] “can produce false positives and false negatives”

Not allowed:

- [ ] “detects all scams”
- [ ] “guarantees safety”
- [ ] “protects your wallet”
- [ ] “prevents wallet theft”
- [ ] “financial advice”
- [ ] “investment advice”

## Link Check

Confirm links are correct:

- [ ] GitHub repo
- [ ] Try Page
- [ ] Examples
- [ ] OpenAPI
- [ ] Live API

## Sensitive Data Check

The article must not contain:

- [ ] password
- [ ] OTP
- [ ] cookie
- [ ] API key
- [ ] GitHub token
- [ ] Cloudflare token
- [ ] private key
- [ ] seed phrase
- [ ] recovery phrase
- [ ] wallet credential
- [ ] personal billing data

## Platform Check

- [ ] Dev.to does not ask for billing/card.
- [ ] Dev.to does not ask for paid upgrade.
- [ ] Dev.to does not ask for API token.
- [ ] Dev.to does not ask for wallet connection.
- [ ] Dev.to content rules are acceptable.

## Final Decision

Choose one:

```text
[ ] Publish manually
[ ] Save as draft
[ ] Stop and review again
```
