# Passive Listing Kit v0.1

## Purpose

This kit prepares NEXUS-X ARIA for safe passive publication.

Passive publication means making the project discoverable without aggressive promotion, spam, direct messaging, forced monetization, or exaggerated claims.

This kit is preparation only. It does not submit ARIA to any platform.

## Safe Positioning

Use this positioning:

```text
NEXUS-X ARIA is a transparent heuristic airdrop risk scoring API for educational and informational use.
```

In Indonesian:

```text
NEXUS-X ARIA adalah API edukatif berbasis heuristik untuk menilai risiko airdrop/Web3 secara transparan.
```

## What ARIA Can Claim

Safe claims:

- heuristic airdrop risk scoring
- educational and informational tool
- transparent rule-based scoring
- returns risk score, risk level, verdict, red flags, safe actions, and summary
- includes Try Page
- includes Examples page
- includes OpenAPI specification
- includes internal test pack
- does not connect to wallets
- does not request seed phrases or private keys

## What ARIA Must Not Claim

Do not claim:

- guaranteed scam detection
- guaranteed wallet protection
- detects all phishing
- prevents wallet theft
- proves an airdrop is safe
- financial advice
- investment advice
- certified security audit
- enterprise-grade security product
- zero false positives
- zero false negatives

## Stop Conditions

Stop immediately if any platform asks for:

- billing
- credit card
- paid upgrade
- payout setup
- API token
- secret
- private key
- seed phrase
- password
- OTP
- cookie
- wallet connection
- sensitive credentials

## Passive Listing Rule

Submit nothing automatically.

Before using any listing text, review:

1. Platform Terms of Service.
2. Category rules.
3. Whether crypto/security/API content is allowed.
4. Whether the platform requires billing.
5. Whether the platform encourages misleading claims.
6. Whether the platform requires monetization.

## Recommended First Public Actions

Recommended order:

1. Keep GitHub repo public and clean.
2. Review README from browser.
3. Publish educational article manually only after account/platform check.
4. Consider one small developer directory only after ToS review.
5. Wait for feedback.
6. Improve docs if needed.
7. Avoid marketplace monetization until abuse controls exist.

## Not Recommended Now

Avoid:

- RapidAPI
- paid API marketplaces
- payout marketplaces
- aggressive marketing
- unsolicited DMs
- paid ads
- fake testimonials
- “guaranteed security” messaging
