# v0.2.1 Parking Lot Review Checklist v0.1

## Review Checklist

Before any idea moves out of parking lot:

```text
[ ] idea has ID
[ ] problem is clear
[ ] evidence exists
[ ] scope is small
[ ] risk is described
[ ] test impact is described
[ ] runtime impact is described
[ ] docs impact is described
[ ] no forbidden credential or billing dependency
```

## Move to Docs-Only Candidate

Allowed when:

```text
[ ] no worker.js change
[ ] no package.json change
[ ] no Cloudflare deploy
[ ] no runtime release
```

## Move to Runtime Candidate

Allowed only when:

```text
[ ] bug or improvement is clear
[ ] test case can be created
[ ] release gate can be satisfied
[ ] rollback is simple
```

## Keep Parked

Keep parked when:

```text
[ ] idea is vague
[ ] evidence is missing
[ ] scope is too large
[ ] requires external services
[ ] requires billing or credentials
```
