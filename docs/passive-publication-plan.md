# Passive Publication Plan

## Goal

Make NEXUS-X ARIA discoverable in a safe, honest, non-aggressive way.

## Current Assets

Already available:

- public GitHub repository
- polished README
- live API landing page
- Try Page
- Examples page
- OpenAPI JSON
- Test Pack
- Publication Kit
- polished Dev.to article draft
- Passive Listing Kit

## Recommended Sequence

### Step 1 — GitHub Repo

Status: done.

Keep repo public and clean.

### Step 2 — README Review

Status: done from browser screenshot.

README is readable and has live links, current status, examples, limitations, and disclaimer.

### Step 3 — Article Review

Review:

```text
docs/devto-article-polished.md
```

Do not publish automatically.

### Step 4 — Manual Platform Review

Before using any platform:

1. Read ToS.
2. Check billing/card requirements.
3. Check category rules.
4. Check crypto/security/API policy.
5. Confirm safe claims.
6. Confirm disclaimer.

### Step 5 — Publish One Thing Only

When ready, publish only one passive item first.

Recommended first candidate:

```text
Dev.to educational article
```

Why:

- educational format
- not marketplace
- no API monetization
- easier to explain limitations

### Step 6 — Wait for Feedback

After publishing one item:

- do not spam
- do not mass-share
- wait for natural feedback
- fix docs if feedback reveals confusion

### Step 7 — Consider One Directory

Only after article/repo looks stable, consider one safe developer directory.

## Not Now

Do not do now:

- RapidAPI
- paid marketplace
- pricing
- subscription
- API key monetization
- paid ads
- mass DMs
- large runtime refactor
