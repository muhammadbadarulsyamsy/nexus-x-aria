# v0.2.1 Idea Evaluation Rubric v0.1

## Purpose

This rubric prevents vague ideas from becoming runtime changes too quickly.

## Evaluation Questions

For each idea, answer:

```text
[ ] What exact problem does this solve?
[ ] Is there evidence from logs, tests, or user feedback?
[ ] Can it be done docs-only?
[ ] Does it require worker.js?
[ ] Does it require package.json?
[ ] Does it require Cloudflare deploy?
[ ] Does it require GitHub runtime release?
[ ] Does it need credentials, API keys, billing, or external services?
[ ] Does it risk overclaiming safety?
[ ] Does it need a test case?
```

## Decision Labels

```text
PARKED
DOCS-ONLY CANDIDATE
RUNTIME CANDIDATE
REJECTED
NEEDS MORE EVIDENCE
```

## Approval Requirements for Runtime Candidate

A runtime idea can only move forward if:

```text
[ ] problem is clearly defined
[ ] expected behavior is defined
[ ] test case can be added
[ ] no credential is needed
[ ] no billing/upgrade is needed
[ ] no external service is needed
[ ] risk of false positives/negatives is considered
[ ] release gate is updated
```

## Default Decision

If unsure:

```text
PARKED
```

Do not code.
