# v0.2.0 Test Plan Draft

## Purpose

This document defines the planned validation for NEXUS-X ARIA v0.2.0.

Current v0.1.9 status:

```text
20 tests
20 passed
0 failed
```

v0.2.0 target:

```text
25 tests
25 passed
0 failed
```

## Existing Tests

Existing 20 tests should remain valid unless scoring logic intentionally changes.

The goal is additive explainability, not major scoring disruption.

## New Tests

### TC21 — score_breakdown exists

Purpose:

```text
Ensure /score-airdrop returns score_breakdown.
```

Expected:

```text
score_breakdown is array
score_breakdown length > 0 when risk rules fire
```

### TC22 — rule_hits include wallet/signing rules

Purpose:

```text
Ensure triggered wallet/signing rules are visible.
```

Input should include:

```text
connect wallet
sign message
```

Expected rule hits:

```text
WALLET_CONNECT_REQUEST
SIGN_MESSAGE_REQUEST
```

### TC23 — input_quality detects missing token_contract

Purpose:

```text
Ensure missing token_contract is reflected in input_quality.
```

Expected:

```text
input_quality.missing_fields includes token_contract
input_quality.level is partial or weak
```

### TC24 — educational approval warning creates false_positive_notes

Purpose:

```text
Ensure educational warnings are not treated as risky approval instructions.
```

Input phrase:

```text
Do not approve unlimited spending.
```

Expected:

```text
false_positive_notes includes educational approval warning note
RISKY_APPROVAL_REQUEST is not triggered for this phrase alone
```

### TC25 — confidence_note exists

Purpose:

```text
Ensure every score response includes heuristic limitation note.
```

Expected:

```text
confidence_note is string
confidence_note mentions heuristic or false positives/false negatives
```
