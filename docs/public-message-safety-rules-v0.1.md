# Public Message Safety Rules v0.1

## Purpose

This document protects NEXUS-X ARIA from unsafe or misleading public claims.

## Allowed Claims

ARIA may be described as:

```text
heuristic
educational
informational
explainable
rule-based
metadata review tool
risk signal scoring API
developer-focused
serverless
open-source
```

## Forbidden Claims

Do not claim:

```text
guaranteed scam detection
protects your wallet
prevents phishing
financial advice
complete security product
detects all scams
verified safety result
wallet protection system
```

## Required Disclaimer

Use this disclaimer or a close variant:

```text
NEXUS-X ARIA is heuristic-only. It does not guarantee safety, does not provide financial advice, and may produce false positives or false negatives.
```

## Forbidden Inputs

Public examples and demos must never ask for real:

```text
private keys
seed phrases
recovery phrases
passwords
OTPs
cookies
API keys
wallet credentials
```

## Safe Example Inputs

Use fake/sample-only values:

```text
https://example.com
0x1234567890abcdef
https://twitter.com/example
Example Airdrop
Base
```

## Public Message Checklist

Before publishing ARIA copy:

```text
[ ] Does not claim guaranteed safety.
[ ] Does not claim financial advice.
[ ] Does not imply wallet protection.
[ ] Mentions heuristic limitation.
[ ] Mentions false positives/false negatives where needed.
[ ] Does not ask for sensitive credentials.
[ ] Describes ARIA as API/risk signal scoring, not a wallet tool.
```
