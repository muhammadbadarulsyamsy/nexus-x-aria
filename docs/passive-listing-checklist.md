# Passive Listing Checklist

Use this checklist before publishing NEXUS-X ARIA on any external platform.

## Must Pass Before Submission

- [ ] Platform Terms of Service reviewed.
- [ ] Platform allows API/security/crypto educational content.
- [ ] Platform does not require billing setup.
- [ ] Platform does not require credit card.
- [ ] Platform does not require paid upgrade.
- [ ] Platform does not require payout setup.
- [ ] Platform does not require API secrets in public form.
- [ ] Listing text does not claim guaranteed scam detection.
- [ ] Listing text does not claim financial advice.
- [ ] Disclaimer is visible.
- [ ] GitHub link is correct.
- [ ] Base URL is correct.
- [ ] `/docs` link is correct.
- [ ] `/try` link is correct.
- [ ] `/examples` link is correct.
- [ ] `/openapi.json` link is correct.
- [ ] `/test-pack` is passing.
- [ ] No seed phrase/private key/token/password appears in submitted material.

## Safe First Channels

- Existing landing page
- GitHub repository
- Dev.to educational article draft
- Personal blog/static page if already free
- Carefully reviewed developer directory

## Use Later

- Postman Public API Network
- GitBook or documentation hub
- Curated Awesome lists
- General API directories after ToS review

## Avoid For Now

- RapidAPI
- Paid API marketplaces
- Payout-based marketplaces
- Platforms that require billing/card
- Platforms that encourage monetization before abuse controls
- Any channel requiring active outreach or DMs

## Stop Conditions

Stop and report before continuing if a platform asks for:

- billing
- credit card
- upgrade plan
- payout setup
- identity document for monetization
- API token
- secret
- private key
- seed phrase
- password
- OTP
- cookie
