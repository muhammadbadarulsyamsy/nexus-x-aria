# Platform Decision Notes

## Use Now

### Existing Landing Page

Already live and free.

Reason:

- no new account
- no billing
- no ToS risk
- already contains docs links

### GitHub Repository

Already public and suitable for passive developer discovery.

Reason:

- public source
- releases
- docs
- issue templates
- safe for passive discovery

### Dev.to Educational Article Draft

Use as education, not promotion.

Reason:

- good for explaining risk scoring
- can link to GitHub and Try Page
- low cost
- passive discovery

## Use Later

### Postman Public API Network

Potentially useful after ToS and workflow review.

Risks:

- account/workspace setup
- collection setup
- visibility settings
- possible plan limitations

### GitBook / Docs Hub

Useful if documentation grows.

Risks:

- some branding/custom features may be paid
- may be redundant while `/docs` exists

### General API Directories

Use one by one after ToS review.

Risks:

- unknown quality
- possible account/billing/backlink requirements
- moderation policies vary

## Avoid For Now

### RapidAPI

Avoid for now.

Reasons:

- marketplace/payout/pricing orientation
- monetization too early
- abuse controls not ready
- possible billing/payment setup

### Paid API Marketplace

Avoid until rate limits, API keys, abuse controls, and pricing model are approved.

### Aggressive Outreach

Do not use:

- DMs
- spam
- mass posting
- unsolicited sales
- direct customer hunting

## Decision Rule

If a platform asks for card, billing, payout, upgrade, token, secret, or sensitive information: stop and report before continuing.
