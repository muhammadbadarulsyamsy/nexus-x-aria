# ARIA Positioning v0.1

## Product Position

NEXUS-X ARIA should be positioned as:

```text
developer-first educational API
```

Not as:

```text
consumer scam detector
wallet security product
financial advice tool
guaranteed anti-phishing system
```

## Product Category

```text
Explainable heuristic risk scoring API
```

## Primary Users

```text
Web3 developers
airdrop dashboard builders
crypto education communities
security education writers
open-source API reviewers
```

## Secondary Users

```text
technical community moderators
open-source builders
students learning Web3 safety signals
```

## Not the Primary User Yet

```text
retail traders
general wallet users
users who want a simple guaranteed safe/scam answer
```

## Core Value

ARIA helps users answer:

```text
What risk signals are present?
Why did the score increase?
Which rules triggered?
What metadata is missing?
What safe actions should be considered?
```

ARIA does not answer:

```text
Is this definitely safe?
Is this definitely a scam?
Should I buy this token?
Should I connect my wallet?
```

## Final Positioning Statement

```text
NEXUS-X ARIA helps developers and educators inspect common airdrop risk signals from project metadata using transparent, explainable heuristic scoring.
```

## NEXUS-X Role

ARIA is the first asset of:

```text
NEXUS-X — Autonomous AI API Asset Foundry
```

Its current role is to prove that NEXUS-X can build, package, publish, document, and validate a lightweight API asset.

Its next challenge is external validation:

```text
traffic
feedback
developer interest
repository engagement
manual publication response
```
