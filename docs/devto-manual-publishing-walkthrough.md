# Dev.to Manual Publishing Walkthrough v0.1

## Purpose

This document explains how to manually publish the NEXUS-X ARIA educational article on Dev.to safely.

This is not an automatic publishing workflow.

Do not share passwords, OTPs, cookies, API keys, GitHub tokens, Cloudflare tokens, private keys, seed phrases, recovery phrases, or wallet credentials with anyone.

## Source Article

Use this file from the repository:

```text
docs/devto-article-polished.md
```

Recommended title:

```text
Transparent Airdrop Risk Scoring: Building a Small Heuristic API for Web3 Safety Education
```

## Before Starting

Confirm:

```text
[ ] You can access your Dev.to account normally.
[ ] Dev.to does not ask for billing/card/paid upgrade for publishing.
[ ] You are not sharing any login credential in chat.
[ ] You are not copying secrets into the article.
[ ] You understand this is educational content, not financial advice.
```

Stop if Dev.to asks for:

```text
billing
credit card
paid upgrade
API token
secret
private key
seed phrase
password in chat
OTP in chat
cookie
wallet connection
```

## Manual Publishing Steps

### Step 1 — Open the Article File

Open this file in GitHub or local editor:

```text
docs/devto-article-polished.md
```

Review the article from top to bottom.

### Step 2 — Copy the Article Content

Copy the full markdown article.

Do not copy:

```text
local Termux paths
credentials
tokens
debug logs
private notes
```

### Step 3 — Open Dev.to

Open Dev.to manually in your browser.

Log in only through the official Dev.to website.

Never send your password or OTP to anyone.

### Step 4 — Create New Post

Create a new post manually.

Paste the article markdown into the editor.

### Step 5 — Add Title

Use the recommended title:

```text
Transparent Airdrop Risk Scoring: Building a Small Heuristic API for Web3 Safety Education
```

### Step 6 — Add Tags

Suggested tags:

```text
web3
security
api
opensource
```

Use fewer tags if the platform limits them.

Avoid tags focused on:

```text
trading
airdrophunting
getrichquick
investment
```

### Step 7 — Preview

Before publishing, use preview mode.

Check:

```text
[ ] headings display correctly
[ ] code blocks display correctly
[ ] links are clickable
[ ] disclaimer appears near top
[ ] limitations section appears
[ ] no credential appears
[ ] tone is educational
```

### Step 8 — Final Safety Check

Before clicking publish, confirm:

```text
[ ] Article does not promise guaranteed scam detection.
[ ] Article does not say ARIA protects wallets.
[ ] Article does not give financial advice.
[ ] Article does not ask users to connect wallets.
[ ] Article does not ask users to enter seed phrases/private keys.
[ ] Article says ARIA is heuristic and can be wrong.
```

### Step 9 — Publish Only If Comfortable

Publish only if everything looks correct.

If unsure, save as draft and stop.

### Step 10 — After Publishing

Save the published URL in local notes.

Do not spam the link.

Do not DM people aggressively.

Do not post in communities that prohibit promotion.

Wait for natural feedback.
