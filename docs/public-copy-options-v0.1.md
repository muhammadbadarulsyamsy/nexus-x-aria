## README Opening

```md
# NEXUS-X ARIA

Transparent heuristic risk scoring for Web3 airdrop metadata.

NEXUS-X ARIA is a lightweight serverless API that helps developers inspect common risk signals in airdrop project metadata. It returns a risk score, risk level, red flags, safe actions, and explainable scoring details such as `rule_hits`, `score_breakdown`, `input_quality`, `confidence_note`, and `false_positive_notes`.

ARIA is designed for developers, educators, and Web3 builders who need a simple way to review airdrop metadata without connecting wallets or handling sensitive credentials.

> Safety note: ARIA is heuristic-only. It does not guarantee safety, does not provide financial advice, and may produce false positives or false negatives. Never enter private keys, seed phrases, recovery phrases, passwords, OTPs, cookies, API keys, or wallet credentials.
```

## Dev.to Opening

```md
I built NEXUS-X ARIA, a lightweight serverless API for explainable Web3 airdrop risk scoring.

The goal is not to claim that an API can guarantee whether an airdrop is safe. Instead, ARIA surfaces common risk signals from project metadata and explains which heuristic rules were triggered.

It returns a risk score, red flags, safe actions, and explainable fields such as `rule_hits`, `score_breakdown`, `input_quality`, and `false_positive_notes`.

The project is open-source, runs as a Cloudflare Worker, includes a browser try page, OpenAPI spec, test pack, examples page, and a local Postman collection.
```
