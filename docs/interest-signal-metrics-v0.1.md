# Interest Signal Metrics v0.1

## Purpose

Define what to measure after manual publication.

The goal is not revenue yet. The goal is signal.

## Primary Signals

```text
GitHub stars
GitHub forks
GitHub issues
GitHub comments/discussions
Dev.to views
Dev.to reactions
Dev.to comments
people asking how to use API
people asking for integration examples
people reporting false positives/false negatives
```

## Secondary Signals

```text
profile visits
social replies
bookmark/saves if visible
Postman collection interest
OpenAPI interest
```

## Negative Signals

```text
no views
no stars
no comments
confusion about what ARIA does
people think it guarantees scam detection
people think it is a wallet tool
```

## Signal Review Window

Recommended first review:

```text
after 7 days
```

Second review:

```text
after 14 days
```

## Minimum Useful Signal

Any of these is useful:

```text
1 meaningful comment
1 feature question
1 integration question
1 GitHub issue
3+ stars from external users
1 external user trying /try or API
```

## Decision After Review

If there is interest:

```text
continue docs and integration examples
consider one small v0.2.1 candidate if feedback is clear
```

If there is no interest:

```text
improve message
try one different channel
or pause ARIA and evaluate next NEXUS-X asset idea
```

If people misunderstand ARIA:

```text
fix positioning and disclaimers before wider distribution
```
