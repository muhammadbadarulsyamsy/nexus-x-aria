# Do Not Use Yet

These actions are not recommended yet.

## Monetization

Do not add yet:

- paid API keys
- subscription plans
- payment links
- payout setup
- marketplace pricing
- paid rate limit tiers

Reason:

ARIA is still MVP and needs abuse control, more validation, and clearer usage feedback.

## Marketplaces

Avoid for now:

- RapidAPI
- paid API marketplaces
- payout-based marketplaces

Reason:

These are too close to monetization and may require billing, payout setup, or abuse management.

## External Services

Do not add yet:

- paid reputation API
- paid domain intelligence API
- paid database
- paid blockchain explorer dependency
- VPS
- managed database

Reason:

The project follows zero/low-cost-first principles.

## Architecture

Do not do yet:

- large refactor
- database migration
- multi-service architecture
- user account system
- secret management complexity

Reason:

The current single-file Worker is simple and stable.

## Promotion

Do not do:

- spam
- mass DM
- unsolicited outreach
- aggressive sales
- misleading claims

Reason:

Publication must remain passive and permission-respecting.
