# NEXUS-X ARIA v0.2 Roadmap Design

This roadmap captures the next hardening phase after v0.1.5.

## Current State

NEXUS-X ARIA v0.1.5 is live, documented, released on GitHub, and has a public OpenAPI spec.

## Main Problems to Address

1. Scoring is still shallow and rule-based.
2. No custom rate limiting or API key system exists.
3. There is no browser form for custom input.
4. No automated CI tests exist.
5. The Worker source is still a single large file.
6. There are limited real-world integration examples.

## Recommended Sequence

### v0.1.6 — Repository Quality Patch

Add repository quality documents and examples without touching the live Cloudflare Worker.

### v0.1.7 — Browser Try Page

Add a `/try` page for custom input testing from a browser.

### v0.1.8 — Scoring Improvement

Improve false-positive handling and expand phrase detection.

### v0.1.9 — Expanded Test Pack

Expand `/test-pack` from 11 tests to at least 25 tests.

### v0.2.0 — Source Structure Refactor

Split the single-file Worker into a maintainable project structure.

### v0.2.1 — GitHub Actions Tests

Add lightweight automated checks for syntax, version, OpenAPI, and test-case consistency.

### v0.3.0 — Abuse Protection

Evaluate rate limiting and optional API key support after reviewing free-tier constraints and operational risk.

## What Not To Do Yet

- Do not monetize.
- Do not add paid API keys.
- Do not claim guaranteed safety.
- Do not add external paid services.
- Do not implement ML before improving heuristics and tests.
- Do not spam or DM potential users.
