# Dev.to Publishing Checklist

Use this before manually publishing the article.

## Account and Platform Safety

- [ ] Dev.to account is accessible.
- [ ] No password, token, OTP, cookie, API key, private key, or seed phrase is shared in chat.
- [ ] Platform does not ask for billing/card/upgrade for normal article publishing.
- [ ] Stop if any unexpected billing, card, or paid upgrade prompt appears.

## Article Safety

- [ ] Article is educational, not aggressive promotion.
- [ ] Article does not claim guaranteed scam detection.
- [ ] Article does not claim ARIA prevents all phishing.
- [ ] Article does not give investment advice.
- [ ] Article does not ask users to connect wallets.
- [ ] Article does not ask users to enter private keys or seed phrases.
- [ ] Article clearly says ARIA is heuristic and can be wrong.
- [ ] Article includes limitations.
- [ ] Article includes safety disclaimer.

## Link Check

- [ ] GitHub link works.
- [ ] Try Page link works.
- [ ] Examples link works.
- [ ] OpenAPI link works.
- [ ] No private/local file path is included.
- [ ] No credential is included.

## Suggested Tags

Use only if appropriate:

```text
web3
security
api
javascript
opensource
```

Avoid tags that may attract speculative or trading-focused audiences.

## Before Clicking Publish

- [ ] Preview article.
- [ ] Check formatting.
- [ ] Check code blocks.
- [ ] Check links.
- [ ] Check disclaimer.
- [ ] Confirm no sensitive data.

## After Publishing

- [ ] Save article URL.
- [ ] Add URL to local notes.
- [ ] Do not spam link in DMs.
- [ ] Do not mass-post aggressively.
- [ ] Consider adding article link to GitHub README later, only if it fits.
